Can I get Arrays!?

What are Arrays?

Arrays are lists
  • All items of the same type
  • Each item has a position
  • There is size limit
They make it easier to declare of a bunch of variables
  • Midterm scores
  • Names of students
  • Classroom numbers
  • ... and much more

Declaring Arrays

  • Arrays are contiguous in memory. they are put in one after another.
  • You can set values like you do for strings using the [ ] operator
  • To manually set initial values, you can initialize within brackets separated by commas.
    {val1, val2, ... }
// declare and initialize an array of ints
int arr[5] = { 16, 2, 77, 40, 12071 }; 
//Set element at index 2
arr[2] = 75;

//get element at index 0
int x = arr[0]

// declaring an array of strings
string coolStrs[3] = {"yo", "sup", "hello"}

The size of the array MUST be constant at compile time.
  • Some computers will compile (Macs do)
  • But many compilers will throw an error (like in visual studio)
#include <iostream>

using namespace std;
int main() {
	int size;
	cin >> size;
	
	// size is not constant. Initializing with constant size will 
	// lead to errors
	int myArr[size];
	
	// if size of myArr is less than 10 BS (BAD STUFF!!) will happen!
	myArr[10] = 999;
}

Arrays and Functions

If you have a function that takes in a variable, you can pass individual elements of arrays. If you pass an element to a reference argument, it will be changed in the function 
#include <iostream>
#include <string>
using namespace std;

// regular copy value parameter
void f1(int i) {
	i = 10;
}

// take reference parameter
void f2(int &i) {
	i = 10;
}

int main () {
	int i[2] = {1, 2};

	f1(i[1]);
	cout << i[1] << endl;
	
	f2(i[1]);
	cout << i[1] << endl;
}
Continuing from the previous example, would this be valid? 
	f1(i);
No, because i is the array. Its not in element. An example error is
candidate function not viable: no known conversion from 'int [3]' to 'int &' for 1st argument

To pass an entire array to a function, you need to use the array parameter 
//Precondition: size is the declared size of the array a. 
//The user will type in size integers.
void print(int arr[], int size) {
	for(int i=0; i < size; i++) {
		cout << arr[i] << endl;
	}
}

The major difference between individual variable parameters and array parameters is that arrays are passed by reference.
The array argument points to the location of the first element. Since the array is contiguous, we can access subsequent elements easily Any changes to arrays in function calls are permanent! 
#include <iostream>
#include <string>
using namespace std;

// passing arrays as const ensures no element can be changed
void print(const int a[], int size) {
	for(int i=0; i < size; i++) {
		cout << a[i] << endl;
	}
}

void init(int a[], int size) {
	for(int i=0; i < size; i++) {
		a[i] = (i+1) * 10;
	}
}

int main () {
	int size = 10;
	int arr[size];
	
	init(arr, size);
	print(arr, size);
}

Inconsistent use of the const parameter will result in errors! 
double computeAverage(int a[], int numberUsed);
//Returns the average of the elements in the first numberUsed 
//elements of the array a. The array a is unchanged.

void showDifference(const int a[],int numberUsed)
{
	double average = computeAverage(a, numberUsed);
	cout << "Average of the " << numberUsed
		<< " numbers = " << average << endl
		<< "The numbers are:\n";
	for (int index = 0; index < numberUsed; index++)
		cout << a[index] << " differs from average by "
		    << (a[index] - average) << endl;
}
Even though computeAveraged does not modify its parameter, the compiler will think that it might try to since the parameter is not read in as const. Since computeAverage is not going to modify the array, declare the parameter as const.
