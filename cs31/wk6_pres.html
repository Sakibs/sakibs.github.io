What do we mean when we say 2D array?
Why would we need 2D arrays?



A 2D array is an array of arrays! You have a number of rows, all with the same number of columns.
Multi dimensional arrays are useful for

  • 2D game maps

2D Arrays as Parameters

2D arrays can be passed to functions and, like regular arrays, they are passed by references so any changes to the array made during function calls are persistent.
What is different is that you can omit the number of rows BUT you MUST specifiy the number of columns

Can anyone see why that is the case? 

 void print(int A[][3],int N, int M);
// ...
// Somewhere in the main

int arr[4][3];
// initialize ...

// call the print function
print(arr, 4, 3);

// Here is a working example
#include <iostream>
using namespace std; 
void print(int A[][3],int N, int M)
{
  for (R = 0; R < N; R++)
    for (C = 0; C < M; C++)
       cout << A[R][C];
}

int main ()
{
  int arr[4][3] ={{12, 29, 11},
                  {25, 25, 13},
                  {24, 64, 67},
                  {11, 18, 14}};
  print(arr,4,3);
  return 0;
}

Now that we looked at arrays, we are going to talk a little bit more about strings
So far we have used C++ strings. If you noticed, you access characters in the string using the [] operator like an array.
That is because C++ strings are very similar to arrays. They are a custom class packed with rich functions

However, before there were C++ strings, there was C++. And before there was C++ there was C. Unfortunately C did not have C++ strings (sad face). It had C-strings.

What are strings composed of?

C-Strings are arrays of characters!

Suppose I want to make a C-String to hold words up to 10 characters like knickknack or lumberjack

How long should I declare this char array variable to be? 

char word[10]; // allocate C-string to hold up to 10 character word
Ok great, so we declared a variable to store all the characters of the longest word. What happens if I put the word "hello" in my variable?

We see that we need a way to indicate until what index is a valid word stored in our C-String variable


How do we signal the end of a C-String? 


char zero_byte = '\0';
We can signal the end of a C-string with a special character called the zero byte '\0'

Going back to the example above, we see that we need one more space to hold the zero byte. So we add 1 to the array size


char word[11]; // allocate C-string to hold up to 10 character word + ending zero byte

What is the benefit of having a dedicated termination character?
